## 0. Experimental Methodology & Measurement Contract

### Measurement Contract
All results represent wall-clock measurements unless otherwise specified. CPU time is reported separately to decouple scheduler noise.

### Reproducibility Constraints
- **Homelab Environment Constraint**: This evaluation was performed on an active homelab environment rather than a clean-room server.
- Background load average (~2.0) and memory pressure (Swap utilization) were present during testing. No static pinning was enforced.
- CPU frequency scaling (Turbo Boost) was explicitly enabled up to 3.5 GHz to reflect real-world deployment behavior.

### Workload Definition
Workload consists of:
- Normalized HTTP query strings
- SecLists adversarial payloads
- URL-encoded mixed encoding ratios (0–100%)

### Statistical Interpretation Rule
Primary metric is the **median**; mean is secondary due to skew sensitivity in tail latency workloads.

### Terminology & Definitions
- **Warm cache**: Repeated invocation with dataset resident in L1/L2 cache.
- **Cold cache**: Dataset forced eviction via controlled memory pressure.
- **Full Stack Overhead**: End-to-end HTTP benchmarks (wrk2) measure full stack overhead including NGINX routing, not pure WAF evaluation cost.
